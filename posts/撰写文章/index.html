撰写文章

徐高亮 Mar 16 2021-03-16T13:37:50+08:00
6 min

原文章地址：https://chirpy.cotes.info/posts/write-a-new-post/

命名与路径

创建一个名为 YYYY-MM-DD-TITLE.EXTENSION 的新文件，并将其放在根目录的 _posts/ 文件夹中。请注意，EXTENSION 必须为 md 或 markdown 。

Front Matter

基本上，您需要在帖子顶部按以下方式填写 Front Matter :

1
2
3
4
5
6

---
title: TITLE
date: YYYY-MM-DD HH:MM:SS +/-TTTT
categories: [TOP_CATEGORIE, SUB_CATEGORIE]
tags: [TAG]     # TAG names should always be lowercase
---

注意: post 的 layout 已经默认设置为 post ，因此无需在 Front Matter 块中添加变量 layout 。

时区

为了准确记录 post 的发布日期，您不仅应设置 _config.yml 中的 timezone，还应在 Front Matter 块的字段 date 中提供 post 的时区。格式: +/-TTTT, 例如 +0800.

类别和标签

每篇文章的 categories 被设计为包含最多两个元素，tags中的元素数量可以是零至无穷大。例如：

1
2

categories: [Animal, Insect]
tags: [bee]

目录

默认情况下，目录（TOC,Table of Contents) 显示在文章右侧面板上，如果要全局关闭它，请转到 _config.yml 并将 toc 变量的值设置为 false。如果要关闭特定文章的目录，请在文章的 Front Matter 中添加以下内容：

1
2
3

---
toc: false
---

评论

与目录相似, Disqus 评论默认情况下会加载到每篇文章中，全局开关由文件 _config.yml 中的变量 comments 定义。如果要关闭特定文章的评论，请将以下内容添加到文章的 Front Matter 中：

1
2
3

---
comments: false
---

数学

出于网站性能的原因，默认情况下不会加载 mathematical 功能。但是可以通过以下方式启用它：

1
2
3

---
math: true
---

Mermaid

Mermaid 是一个很棒的图表生成工具。要在您的帖子中启用它，请将以下内容添加到YAML块中：

1
2
3

---
mermaid: true
---

然后，您可以像其他 markdown 语言一样使用它: 用 ```mermaid 和 ``` 包围图形代码。

图片

预览图片

如果要将图像添加到帖子内容的顶部，请指定图片的 url 和 alt 属性：

1
2
3
4
5

---
image:
  src: /path/to/image/file
  alt: image alternative text
---

图片说明

将斜体添加到图像的下一行，然后它将成为标题并显示在图像的底部：

1
2

![img-description](/path/to/image)
_Image Caption_

图片大小

您可以使用 width 命令指定图像的宽度（和高度）:

1

![Desktop View](/assets/img/sample/mockup.png){: width="400"}

图片位置

默认情况下，图像居中，但你可以通过使用类别normal , left 和 right 中的一个指定位置。例如：

  • 正常位置

    图片将在以下示例中保持对齐：

    1

    ![Desktop View](/assets/img/sample/mockup.png){: .normal}

  • 向左浮动

    1

    ![Desktop View](/assets/img/sample/mockup.png){: .left}

  • 向右浮动

    1

    ![Desktop View](/assets/img/sample/mockup.png){: .right}

限制: 一旦指定了图像的位置，就禁止添加图片说明。

CDN URL

如果你把图片放在了 CDN 上，你可以通过在 _config.yml 文件中分配变量 img_cdn 来节省重复编写 CDN url的时间：

1

img_cdn: https://cdn.com

一旦 img_cdn 被分配, CDN url 将被添加到以 / 开始的所有图片（avatar 和 posts的图片）路径上。

例如，当使用图片时：

1

![The flower](/path/to/flower.png)

解析结果将自动在图片路径之前添加CDN前缀 https://cdn.com：

1

<img src="https://cdn.com/path/to/flower.png" alt="The flower">

已标记文章

您可以将一个或多个文章固定在主页顶部，固定文章根据其发布日期以相反的顺序排序。通过以下方式启用：

1
2
3

---
pin: true
---

代码块

Markdown 符号 ``` 可以轻松创建代码块，如以下示例所示。

1

This is a common code snippet, without syntax highlight and line number.

特定语言

使用 ```language 您将获得带有行号和语法突出显示的代码段。

注意: Jekyll style {% highlight LANGUAGE %} or {% highlight LANGUAGE linenos %} 不允许在此主题中使用！

1
2
3
4
5
6

# Yaml code snippet
items:
    - part_no:   A4786
      descrip:   Water Bucket (Filled)
      price:     1.47
      quantity:  4

Liquid 代码

如果要显示 Liquid 代码段, 请用 {% raw %} 和 {% endraw %} 包围 Liquid 代码。

1
2
3

{% if product.title contains 'Pack' %}
  This product's title contains the word Pack.
{% endif %}
